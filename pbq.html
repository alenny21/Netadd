PBQ Practice Scenarios

Beginner

No Internet Access

A user cannot access the internet. They can connect to Wi-Fi, but websites do not load.

What should you check first?
Start simple:
  1. Check IP address
  2. Check default gateway
  3. Check DNS server
  4. Ping gateway
  5. Ping 8.8.8.8
  6. Test DNS with nslookup
ipconfig /all
ping 8.8.8.8
nslookup google.com
Beginner

APIPA Address

A workstation received 169.254.44.10

What most likely failed?
DHCP failed.

The computer gave itself an APIPA address because it could not reach the DHCP server.

  1. Check DHCP server
  2. Check switch/VLAN
  3. Check cable or Wi-Fi
  4. Run ipconfig /renew
Intermediate

Switch Loop

Users report extremely slow network speeds after a second switch was connected.

What protocol likely helps prevent this issue?
STP (Spanning Tree Protocol)

STP prevents Layer 2 loops by blocking redundant switch paths.

  1. Check STP status
  2. Look for redundant paths
  3. Check BPDU Guard
  4. Verify switch topology
Intermediate

Subnetting PBQ

You need at least 30 usable hosts.

Which subnet should you use?
/27

/27 gives:

  • 32 total addresses
  • 30 usable hosts

Remember: 128 → 64 → 32 → 16 → 8 → 4

Intermediate

DNS Failure

Users can ping IP addresses, but cannot open websites using names.

What service is most likely failing?
DNS

If IP connectivity works but names fail, DNS is usually the problem.

nslookup google.com
Intermediate

Remote Access

A technician needs secure remote CLI access to a Linux server.

Which protocol and port should be used?
SSH — Port 22

SSH securely encrypts remote terminal sessions.

Avoid Telnet because it is insecure.
