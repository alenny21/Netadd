Troubleshooting Commands

Stop memorizing commands blindly.

Learn what each command is actually checking. Most Network+ troubleshooting questions become easier once you know the purpose behind the tool.

Windows Commands

ipconfig

ipconfig /all

Shows IP address, subnet mask, gateway, DNS server, MAC address, and DHCP information.

Think: “What network settings does this device currently have?”
IP Troubleshooting

ping

ping 8.8.8.8

Tests whether another device responds over the network using ICMP.

Start simple. Ping checks basic connectivity first.
Connectivity

tracert

tracert google.com

Shows the path packets take across routers to reach a destination.

Think: “Where along the path is traffic slowing or failing?”
Routing

nslookup

nslookup google.com

Checks DNS name resolution.

If ping by IP works but names fail → think DNS.
DNS

netstat

netstat -an

Displays active connections and listening ports.

Great for checking suspicious connections or open ports.
Connections

arp

arp -a

Displays the ARP table showing IP-to-MAC mappings.

ARP connects Layer 3 IPs to Layer 2 MAC addresses.
Layer 2

Cisco Commands

show ip route

show ip route

Displays the routing table.

Think: “How does this router know where to send traffic?”
Routing

show vlan brief

show vlan brief

Displays VLANs and assigned switch ports.

Great for checking whether devices are in the wrong VLAN.
Switching

show interfaces

show interfaces

Displays interface status, speed, duplex, errors, and statistics.

Use this for duplex mismatches and interface errors.
Interfaces

show mac address-table

show mac address-table

Displays learned MAC addresses and switch ports.

Helps track where a device physically connects.
Layer 2

Exam Strategy

No Internet?

Start with: ipconfig → ping → nslookup

Switch Problem?

Think: VLANs, interfaces, MAC table

Routing Problem?

Think: show ip route

DNS Problem?

If IP works but names fail: nslookup
