Port Strategy Group ports by purpose instead of memorizing random numbers.

Common Ports

Learn the story, not the number.

Most Network+ port questions become easier when you group protocols by purpose.

Remote Access

22 — SSH

Secure remote access.

Memory trick: 22 comes before insecure 23.

23 — Telnet

Old insecure remote terminal.

No encryption.

3389 — RDP

Remote Desktop Protocol.

Used for Windows remote desktop access.

Web Traffic

80 — HTTP

Regular web traffic.

443 — HTTPS

Secure encrypted web traffic.

Memory trick: Bigger safer number.

File Transfer

Port Protocol Purpose
20/21 FTP Old file transfer
22 SFTP Secure file transfer over SSH
69 TFTP Simple lightweight file transfer

Memory Trick

FTP is old. SFTP upgraded to secure SSH port 22. TFTP is tiny/simple FTP.

Network Services

Port Protocol Purpose
53 DNS Name resolution
67/68 DHCP Automatic IP addressing
123 NTP Time synchronization
161 SNMP Network monitoring

53 — DNS

Converts names into IP addresses.

67/68 — DHCP

Two ports because server and client both communicate.

123 — NTP

Memory trick: 1-2-3 time.

161 — SNMP

Used for monitoring and management.

Authentication + Security

Port Protocol Purpose
49 TACACS+ Cisco AAA authentication
88 Kerberos Authentication
1812 RADIUS Authentication
1813 RADIUS Accounting Accounting logs

What Actually Matters for the Exam

Know the common ones cold

20/21, 22, 23, 25, 53, 67/68, 80, 110, 123, 143, 161, 389, 443, 3389.

Focus on purpose

The exam usually tests what the protocol DOES, not just the number.

Security matters

Know which protocols are secure vs insecure: SSH vs Telnet, HTTPS vs HTTP, SFTP vs FTP.

Group by category

Remote access, web, file transfer, and network services are easier to remember than giant random lists.
