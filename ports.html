Port Rule Group ports by purpose. Do not memorize random numbers.

Common Ports

Turn ports into stories.

Ports are easier when grouped by what they do: remote access, web, file transfer, network services, email, and security.

Must-Know First

22 SSH / SFTP
53 DNS
80 / 443 HTTP / HTTPS
67 / 68 DHCP

Grouped Port Review

Remote Access
22 is secure. 23 is old and insecure. 3389 is remote desktop.
PortProtocolUse
22SSH / SFTPSecure remote command line and secure file transfer
23TelnetInsecure remote command line
3389RDPRemote Desktop Protocol
Web
HTTPS is safer and bigger, so it has the bigger number: 443.
PortProtocolUse
80HTTPNormal web traffic
443HTTPSSecure web traffic
File Transfer
FTP is old. SFTP rides on SSH port 22. TFTP is tiny/simple.
PortProtocolUse
20 / 21FTPFile transfer data and control
22SFTPSecure file transfer over SSH
69TFTPSimple file transfer, often for device configs
Network Services
NTP is 123 because clocks go 1-2-3.
PortProtocolUse
53DNSNames to IP addresses
67 / 68DHCPAutomatic IP address assignment
123NTPTime synchronization
161 / 162SNMPNetwork monitoring and traps
514SyslogLog collection
Email
SMTP sends. POP3 and IMAP receive.
PortProtocolUse
25SMTPEmail sending
110POP3Email retrieval
143IMAPEmail retrieval and sync
587SMTPSSecure email submission
993IMAPSSecure IMAP
995POP3SSecure POP3
Directory, Database, and Voice
LDAP is directory services. RDP is remote desktop. SIP starts VoIP calls.
PortProtocolUse
389LDAPDirectory services
636LDAPSSecure directory services
445SMBWindows file and printer sharing
1433MS SQLMicrosoft SQL Server
5060 / 5061SIPVoIP signaling
