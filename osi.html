OSI Model

Use it like a checklist.

The OSI model is not just something to memorize. It helps you troubleshoot from physical problems up to application problems.

Memory Trick

Top to Bottom

All People Seem To Need Data Processing

Application, Presentation, Session, Transport, Network, Data Link, Physical.

Bottom to Top

The 7 Layers

7

Application

The layer users interact with.

Examples: HTTP, HTTPS, DNS, SMTP
6

Presentation

Formats, translates, compresses, or encrypts data.

Examples: TLS, SSL, encryption, compression
5

Session

Starts, manages, and ends communication sessions.

Think: keeping conversations open
4

Transport

Handles ports, reliability, and delivery methods.

Examples: TCP, UDP, port numbers
3

Network

Handles IP addressing and routing between networks.

Examples: IP, routers, ICMP
2

Data Link

Handles MAC addresses, frames, switches, and VLANs.

Examples: Ethernet, MAC, switch, ARP
1

Physical

Handles cables, signals, ports, light, electricity, and hardware.

Examples: cables, connectors, NICs, link lights

Troubleshooting With OSI

Layer 1: Physical

Check cable, Wi-Fi connection, link lights, NIC, power, and physical damage.

Layer 2: Data Link

Check switch port, VLAN, MAC table, duplex mismatch, and STP issues.

Layer 3: Network

Check IP address, subnet mask, default gateway, routing table, and APIPA.

Layer 4: Transport

Check ports, TCP/UDP, firewall rules, and whether the service is listening.

Layers 5–7

Check sessions, encryption, DNS names, application settings, browser, or service issues.

Exam Move

If the question says cable, signal, link light, or connector, think Layer 1 first.
